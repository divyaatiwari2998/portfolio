Project Overview

What This Project Solved

Retail businesses often drown in transaction data without a clear way to act on it. This project tackled exactly that — turning 8,523 raw retail transactions into a clear, interactive business intelligence solution that non-technical stakeholders could use confidently.

The core deliverable was a Power BI dashboard that centralised all KPIs, removed the need for scattered Excel files, and made performance gaps visible at a glance. The project covered customer behaviour trends, outlet-level performance, and product category analysis.

Methodology

How It Was Built

01
Data Collection & Cleaning
Ingested 8,523 retail transaction records. Handled missing values, standardised outlet and product category fields, and validated date formats.
02
Exploratory Data Analysis
Identified sales patterns, seasonal trends, and outlet-level performance gaps using summary statistics and distribution analysis.
03
Dashboard Design
Structured the Power BI report with a clean UX: a KPI overview page, drill-through outlet view, and product category breakdown with slicers.
04
Insights & Recommendations
Translated visual findings into actionable business recommendations for inventory prioritisation, high-performing outlet strategies, and customer engagement.

Key Findings

What the Data Revealed

🏆
High-performing outlet segments showed 2–3× revenue compared to low-performing segments, pointing to clear operational patterns worth replicating.
📦
Top product categories drove a disproportionate share of revenue, enabling targeted inventory prioritisation that could reduce holding costs significantly.
👤
Customer purchase patterns revealed distinct behaviour trends by outlet type, supporting tailored promotions and stock planning strategies.
⚡
Centralised reporting via the Power BI dashboard reduced manual analysis effort substantially, enabling faster decisions without dependency on spreadsheets.
